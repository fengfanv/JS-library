Snowflake Instead of repeating the same code over and over again we can also create a definition for a shape and reuse it. Here we define a branch of a snowflake then use it six times with different rotations. The branch is defined as a path. The path is the most powerful SVG tag. We can define pretty much anything with paths and if you open any SVG file, you will see paths mostly. The shape of the path is defined by the d attribute. Here we define several drawing commands. A command always starts with a letter defining the command type and ends with a coordinate. Here we only have the two most simple commands, move to and line to. The move to command moves the cursor to a point without drawing a line and the line to command draws a straight line from the previous point. A command always continues the previous command so when we draw a line we only define the endpoint. The starting point will be the previous command’s endpoint. This path is a bit unusual because there are several move to commands in it to draw the main branch and each side branches with the same path.

雪花 我们也可以为形状创建一个定义并重用它，而不是一遍又一遍地重复相同的代码。在这里，我们定义了雪花的一个分支，然后以不同的旋转方式使用它六次。 分支被定义为路径。路径是最强大的SVG标签。我们几乎可以用路径定义任何东西，如果你打开任何SVG文件，你会看到大部分路径。 路径的形状由d属性定义。在这里，我们定义了几个绘图命令。命令总是以定义命令类型的字母开头，以坐标结尾。这里我们只有两个最简单的命令，move to和line to。move to命令将光标移动到一个点而不绘制线，line to命令从前一个点绘制一条直线。命令总是延续前一个命令，所以当我们画一条线时，我们只定义端点。起点将是前一个命令的终点。此路径有点不寻常，因为其中有几个move-to命令，用于绘制具有相同路径的主分支和每个侧分支。
